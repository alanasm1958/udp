## Animation Guidelines

### Transitions
- **Card hover**: `transform: scale(1.02)` + shadow increase
- **Drawer open**: Slide in from right (300ms ease-out)
- **Modal open**: Fade in + scale up (200ms ease-out)
- **Button click**: Scale down to 0.95 (100ms)

### Loading States
- Skeleton loaders for tables
- Spinner for async operations
- Progress bar for multi-step processes

---

## Accessibility

### Keyboard Navigation
- Tab order follows visual flow
- Enter to activate buttons/links
- Escape to close modals/drawers
- Arrow keys for table navigation

### Screen Readers
- Semantic HTML (heading hierarchy)
- ARIA labels for icons
- Status announcements for changes
- Form field labels and descriptions

### Color Contrast
- Minimum 4.5:1 for normal text
- 3:1 for large text
- Status not conveyed by color alone (use icons)

---

## Best Practices

### Forms
1. Required fields marked with *
2. Inline validation (on blur)
3. Clear error messages
4. Disable submit until valid
5. Show loading state on submit
6. Success feedback after save

### Tables
1. Sortable columns (where applicable)
2. Filterable by common attributes
3. Searchable by key fields
4. Paginated (50 items default)
5. Row actions in menu (â‹®)
6. Bulk actions when relevant

### Drawers/Modals
1. Clear title
2. Close button (X)
3. Back navigation for nested views
4. Sticky footer for actions
5. Scroll content area only
6. Confirm before closing unsaved changes
